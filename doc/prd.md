## 1. 引言 / 概述

Mac Super Assistant 是一款 macOS 原生效率工具，定位为 **增强版 Spotlight + 常用效率工具集成中心**。

产品通过一个统一搜索入口，将应用启动、剪贴板历史、截图、内置命令、计算器、设置等高频能力聚合在一起。同时，对于需要更复杂交互的能力，提供轻量管理中心或专用面板承载。

产品第一版不是内部 Demo，而是 **小范围公开测试级 MVP（public beta-ready MVP）**：功能深度保持克制，产品体验、权限引导、隐私说明、设置、稳定性和发布资料按公开产品方向设计；Developer ID 签名和 Apple Notarization 公证作为 MVP 后、公开扩大分发前的发布增强项。

## 2. 产品定位

### 2.0 命名策略

- 产品暂定名：Mac Super Assistant。
- 工程/内部代号：Assistant。
- 正式发布前再确定最终品牌名称、图标和对外文案。

### 2.1 核心定位

产品核心定位为：

> 以 Spotlight 式统一搜索入口为主，以截图、剪贴板、命令、应用启动等工具能力为支撑的 macOS 效率中心。

产品不是单纯的应用启动器，也不是松散的工具箱，而是：

- **搜索入口优先**：所有核心功能都应能通过搜索框触发。
- **独立面板补充**：剪贴板历史、设置、权限等低频/复杂操作可通过管理中心完成。
- **开箱即用**：默认启用主要能力，减少用户前置配置和认知负担。
- **本地优先**：剪贴板、截图、配置等数据默认只保存在用户本机。

### 2.2 与现有工具的关系

产品目标不是完整复制某一个工具，而是覆盖以下工具的高频场景：

| 工具类型        | 代表产品                              | 本产品覆盖方向                 |
| ----------- | --------------------------------- | ----------------------- |
| 应用启动 / 命令入口 | Spotlight / Alfred / Raycast      | 统一搜索框、应用启动、内置命令、计算器     |
| 剪贴板管理       | Maccy / Paste                     | 文本、图片、文件剪贴板历史           |
| 截图工具        | iShot / Shottr / CleanShot X      | 区域/全屏/窗口截图，轻量标注，复制/保存   |
| 系统效率工具      | Magnet / iStat Menus / CleanMyMac | 作为长期功能方向：窗口控制、资源监控、应用卸载 |

## 3. 目标

### 3.1 产品目标

- 提供一个统一入口，让用户通过快捷键快速完成常用 macOS 操作。
- 在 MVP 中支持应用启动、剪贴板历史、基础截图、轻量标注和内置命令。
- 以可公开发布产品的标准交付，包含首次引导、权限管理、隐私说明、设置页面和清晰菜单栏入口。
- 默认开箱即用，减少用户在安装后的配置负担。
- 所有用户数据默认仅保存在本地，不上传、不同步、不训练。

### 3.2 MVP 原则

MVP 采用以下原则：

1. **功能深度按演示级克制**：不追求每个模块都完整替代专业工具。
2. **交付质量按公开发布级要求**：首次引导、权限、设置、隐私、错误状态必须清晰。
3. **统一入口优先**：核心功能必须能通过搜索框触发。
4. **Provider 架构优先**：搜索结果来源从第一天按 SearchSource / Provider 模式设计。
5. **默认开启，显式告知**：尽量减少配置，但在 Onboarding 和隐私说明中明确告知默认行为。

## 4. 用户画像

### 4.1 普通办公人员

需求：

- 快速启动常用应用。
- 复制过的内容可以找回。
- 快速截图并标注后发送。
- 不想学习复杂命令和规则。

### 4.2 产品经理 / 运营 / 设计协作用户

需求：

- 高频截图。
- 截图后添加矩形框、箭头、文字、马赛克。
- 截图后快速复制到聊天工具或保存文件。
- 管理最近复制过的文本、链接、图片和文件。

### 4.3 程序员 / 高级用户

需求：

- 快速启动开发工具。
- 快速找回复制过的代码片段、链接、路径。
- 通过内置命令完成常用系统操作。
- 希望长期支持更多高级能力，例如文件搜索、窗口控制、资源监控、自定义命令等。

## 5. 产品范围

### 5.1 MVP 必须包含

MVP 功能集：

1. **统一搜索入口**
   - 默认快捷键 `⌥ Space` 呼出。
   - Spotlight 外观 + Raycast 行为模型。
   - 支持结果列表、图标、类型、说明、键盘选择、回车执行。

2. **应用启动**
   - 搜索并启动常规应用目录中的 `.app`。
   - MVP 搜索范围：`/Applications`、`~/Applications`、`/System/Applications`。
   - 长期目标：支持更广泛应用索引，包括所有 `.app`、系统设置项、命令行工具、浏览器 PWA、Xcode 模拟器 App 等。

3. **剪贴板历史**
   - 默认开启记录。
   - 支持文本、图片、文件三类内容。
   - 文件只保存引用/路径，不复制文件内容。
   - 只按时间淘汰，默认保留最近 30 天。
   - 不做敏感内容过滤，因为数据仅本地保存。

4. **截图系统**
   - 支持区域截图、全屏截图、窗口截图。
   - 截图后显示预览和工具栏。
   - 工具栏提供复制、保存、标注工具、取消。
   - 标注工具支持矩形框、箭头、文字、马赛克/模糊、撤销/重做。

5. **内置命令执行**
   - 只支持内置白名单命令。
   - 不支持搜索框直接执行任意 shell。
   - MVP 不支持用户自定义 shell 命令。

6. **管理中心**
   - 首页 / 概览。
   - 剪贴板历史。
   - 设置。
   - 权限。

7. **菜单栏 App 形态**
   - App 常驻菜单栏。
   - 默认无 Dock 图标。
   - 菜单栏入口包含：打开搜索、剪贴板、截图、设置、关于、退出。

8. **首次启动 Onboarding**
   - 完整首次引导。
   - 强制完成 MVP 必要权限。
   - 如果用户拒绝必要权限，则停留在权限引导页，不能进入完整产品体验。

### 5.2 MVP 搜索源

MVP 搜索源包含：

- 应用（Applications Source）
- 剪贴板（Clipboard Source）
- 内置命令 / 截图命令（Command Source）
- 计算器 / 单位换算（Calculator / Converter Source）
- 设置项（Settings Source）

搜索源配置：

- MVP 的搜索源配置只提供开关级能力。
- 开关语义为：是否参与搜索结果展示。
- 对剪贴板这类隐私敏感功能，额外提供“启用剪贴板历史记录”开关。
- 另提供具体搜索结果黑名单，用于隐藏某个 App、某条内置命令、某个设置入口等具体结果；黑名单不等同于停用功能模块。

### 5.3 MVP 不包含

MVP 不包含以下内容：

- 付费限制、支付系统、授权码、订阅、账号系统。
- 账号登录、用户注册、云端账户体系。

- 任意 shell 执行。
- 用户自定义命令。
- 插件市场或第三方插件系统。
- 文件全文搜索 / 文件索引。
- OCR。
- 录屏。
- 长截图。
- 截图贴图。
- 截图历史管理。
- 云同步。
- AI 能力。
- 应用卸载。
- 资源监控。
- 窗口控制完整功能。
- 多设备同步。
- 团队协作。

### 5.4 长期功能地图

PRD 需要记录完整愿景，但开发按 MVP 分期。长期功能包括：

- 文件搜索。
- 窗口控制。
- 资源监控。
- 应用卸载。
- OCR。
- 录屏 / GIF。
- 长截图。
- 截图历史。
- 自定义命令。
- 工作流。
- 插件或扩展能力。
- AI 总结、翻译、智能搜索。

### US-003：统一搜索入口

**描述：** 作为用户，我想通过 `⌥ Space` 打开一个统一搜索框，以便快速启动应用、查找剪贴板、执行命令或打开设置。

### US-004：SearchSource Provider 架构

**描述：** 作为开发者，我需要搜索源以 Provider 模式组织，以便后续扩展文件、窗口、资源监控、应用卸载等来源。

**验收标准：**

- [ ] 搜索源抽象为 SearchSource / Provider。
- [ ] 每个 Provider 有唯一 id、启用状态、搜索能力和执行能力。

- [ ] 设置中的搜索源开关能控制对应 Provider 是否参与搜索展示。

### US-005：搜索结果多动作模型

**描述：** 作为开发者，我希望搜索结果从一开始支持主动作和次级动作，以便后续增加更多操作时不重构核心模型。

**验收标准：**

- [ ] SearchResult 支持 primaryAction。
- [ ] SearchResult 支持 secondaryActions。
- [ ] MVP UI 只执行 primaryAction。
- [ ] MVP 不要求展示 Cmd+K 动作面板。
- [ ] 后续可基于 secondaryActions 增加右键菜单或动作面板。

## 7. 功能需求

### 7.1 搜索入口

- FR-SEARCH-1：系统必须提供一个全局搜索入口。
- FR-SEARCH-2：默认全局快捷键必须为 `⌥ Space`。
- FR-SEARCH-3：用户必须可以在设置中修改全局快捷键。
- FR-SEARCH-4：搜索框必须支持结果列表展示。
- FR-SEARCH-5：结果项必须包含图标、标题、类型和说明。
- FR-SEARCH-6：回车必须执行当前选中结果的主动作。
- FR-SEARCH-7：ESC 必须关闭搜索框。
- FR-SEARCH-8：搜索源必须可通过设置开关控制是否展示。
- FR-SEARCH-8a：所有 MVP 搜索源默认必须开启，包括应用、剪贴板、内置命令/截图命令、计算器/单位换算、设置项。
- FR-SEARCH-8b：MVP 只有剪贴板提供功能启用/停用开关；其他功能模块不提供独立功能开关。
- FR-SEARCH-8c：搜索结果必须支持黑名单机制，黑名单内的结果不予展示。
- FR-SEARCH-8d：搜索黑名单用于隐藏用户不希望看到的具体结果，不等同于停用功能模块。
- FR-SEARCH-8e：MVP 搜索黑名单粒度只支持屏蔽具体搜索结果。
- FR-SEARCH-8f：MVP 黑名单可用于隐藏某个 App、某条内置命令、某个设置入口等具体结果。
- FR-SEARCH-8g：MVP 不支持规则黑名单，例如按关键词、路径、来源规则批量隐藏。
- FR-SEARCH-8h：MVP 不实现搜索结果右键菜单或完整次级动作入口。
- FR-SEARCH-8i：MVP 搜索黑名单只能通过设置页管理，允许查看、添加和移除已隐藏结果。
- FR-SEARCH-8j：黑名单移除后，对应搜索结果必须可以重新展示。
- FR-SEARCH-8k：从搜索结果右键菜单或次级动作加入黑名单作为后续版本能力预留。

## 10. 技术考量

### 10.1 技术栈

MVP 技术栈确定为：

- SwiftUI。
- AppKit。
- 原生 macOS API。

选择原因：

- 搜索框、菜单栏、截图、剪贴板、权限、应用启动都依赖 macOS 原生能力。
- 原生实现可以获得更好的性能和系统体验。
- 可公开发布时，签名、公证、权限管理更符合 Apple 生态。
- MVP 不需要跨平台。

### 10.2 核心架构

系统采用 SearchSource / Provider 模式：

```swift
protocol SearchSource {
    var id: String { get }
    var isEnabled: Bool { get }
    func search(query: String) async -> [SearchResult]
}

struct SearchResult {
    let title: String
    let subtitle: String
    let primaryAction: SearchAction
    let secondaryActions: [SearchAction]
}
```

说明：

- Provider 是内置搜索源，不等同于第三方插件系统。
- MVP 不做插件市场、热加载或外部扩展。
- 多动作模型从数据层预留，UI 第一版只执行主动作。

### 10.3 隐私与数据策略

- 所有剪贴板历史仅本地存储。
- 截图和图片数据仅本地处理。
- 不上传用户数据。
- 不同步用户数据。
- 不用用户数据训练模型。
- 不做敏感内容过滤。
- 通过 Onboarding 和隐私说明明确告知以上行为。

### 10.4 权限策略

- 采用完整 Onboarding。
- MVP 必要权限在首次启动阶段强制完成。
- 用户拒绝必要权限时不能进入完整产品体验。
- 权限页必须解释用途，并提供打开系统设置与重新检测。
- 除屏幕录制和辅助功能这两个已明确列为 MVP 强制权限的权限外，其他非 MVP 权限不应提前请求。

### 10.5 数据存储技术选型

MVP 数据存储选型为：

- Core Data。
- 文件系统。

存储职责划分：

- Core Data 存储剪贴板历史元数据、文本内容、富文本索引信息、设置项、搜索黑名单、使用统计、置顶状态、保留时间等结构化数据。
- 文件系统存储图片原图、图片缩略图、RTF/HTML 原始格式数据等较大对象。
- Core Data 记录文件系统资源路径或标识，用于关联历史项与实际文件资源。
- 大对象文件按内容类型分目录存储，MVP 至少包含 `Clipboard/Images/`、`Clipboard/Thumbnails/`、`Clipboard/RichText/`。

选择原因：

- Core Data 是 Apple 原生持久化方案，适合纯 SwiftUI + AppKit 的 macOS 原生应用。
- 与 Apple 生态集成更自然，便于后续使用系统能力和迁移。
- 大对象不直接塞入数据库，避免数据库膨胀；图片和富文本原始数据优先放在文件系统。

MVP 不采用：

- SQLite/GRDB 作为主存储。
- 纯 JSON 文件存储。

### 10.6 搜索索引策略

MVP 采用 Core Data 持久化 + 轻量内存搜索索引：

- Core Data 负责持久化结构化数据和详情记录。
- 内存索引用于统一搜索入口和剪贴板历史页面的快速查询。
- 内存索引至少包含：记录 id、标题/文本摘要、内容类型、拼音/别名字段、更新时间、置顶状态、来源类型、使用统计等排序所需字段。
- App 启动时从 Core Data 全量加载必要的轻量索引字段。
- 内存索引不加载图片原图、RTF/HTML 原始数据等大对象。
- 剪贴板数据新增、更新、删除、置顶变化时，必须同步更新内存索引。
- 搜索命中后再按需从 Core Data 和文件系统加载详情或大对象。

MVP 不采用：

- SQLite FTS5。
- Spotlight/系统索引作为剪贴板搜索主路径。

## 11. 非功能需求

### 11.1 性能

- 冷启动目标：≤ 1 秒。
- 搜索结果响应目标：≤ 100ms。
- 后台空闲 CPU：≤ 1%。
- 后台常驻内存目标：≤ 150MB，理想 ≤ 100MB。

### 11.2 可用性

- 用户首次完成 Onboarding 后，应无需额外配置即可使用搜索、剪贴板、截图和内置命令。
- 搜索入口必须稳定响应全局快捷键。
- 菜单栏必须始终可作为找回入口。

### 11.3 稳定性

- 权限缺失时必须给出明确提示。
- 截图失败时必须有错误提示和恢复路径。
- 剪贴板记录失败时不得导致 App 崩溃。
- 文件剪贴板路径失效时必须显示“文件已移动或删除”一类状态。

### 11.4 发布要求

- MVP 阶段暂不强制要求应用签名和 Apple 公证，优先完成可用产品闭环。
- MVP 后、公开扩大分发前，需要补齐 Developer ID 签名和 Apple Notarization 公证流程。
- 需要提供隐私说明。
- 需要提供关于页。
- 需要提供退出 App 能力。

   - 黑名单过滤。
   - 最近使用加权。
   - Core Data 与文件系统资源路径关联。

3. 手动验证清单
   - 首次 Onboarding 流程。
   - 权限授权、拒绝和重新检测。
   - 菜单栏入口。
   - 搜索框打开、失焦关闭、ESC、再次快捷键关闭。
   - 应用启动。
   - 剪贴板文本、富文本、图片、文件记录与恢复。

   - 设置页配置项。
   - 关于页、隐私政策、反馈邮件、检查更新。

4. 权限验证
   - 屏幕录制权限未授权时阻止完成 Onboarding。
   - 辅助功能权限未授权时阻止完成 Onboarding。

   - 默认跟随系统语言。
   - 设置中切换语言可用，必要时提示重启生效。

6. 性能验证
   - 冷启动目标 ≤ 1 秒。
   - 搜索响应目标 ≤ 100ms。
   - 后台空闲 CPU ≤ 1%。
   - 后台常驻内存目标 ≤ 150MB，理想 ≤ 100MB。

后续开发阶段应将上述策略同步到 `doc/test.md`，并按用户故事拆分为可执行测试清单。

MVP 成功标准：

- 用户完成首次启动引导后，可以通过 `⌥ Space` 打开搜索框。
- 用户可以在搜索框内启动常规应用目录中的 App。

- App 能以菜单栏常驻方式稳定运行。
- 所有剪贴板和截图数据均只保存在本地。
